# Planning Skill Protocol

Plan Pilot 使用内置 Planning Skill Protocol 约束规划访谈。它不是依赖某一家大模型的临时提示词，而是前端调用模型时统一加载的规划规则。

## 设计目标

- 已经明确的事项先落地，缺失信息后追问。
- 固定安排、目标和任务使用结构化数据，不依赖模型的口头承诺。
- 追问不能阻止当前轮已经明确的固定安排进入日程。
- 同一套协议可用于 DeepSeek、OpenAI-compatible 服务和 Anthropic Messages 接口。

## 每轮流程

1. 从晨间规划和用户消息中本地提取带时间的会议、监考、出行等固定安排。
2. 立即将明确固定安排写入不可用时间块。
3. 将目标、任务、时间块和访谈上下文发送给当前模型。
4. 要求模型每轮只返回 JSON，包括阶段、说明、目标、任务和固定安排。
5. 本地再次去重并过滤冲突，再展示可加入计划的建议。

## 为什么暂不直接使用 MCP

MCP 更适合后续连接外部日历、提醒服务、文件和知识库。当前原型的核心问题是规划行为一致性，因此第一版使用项目内置协议更轻量，也能覆盖不同模型供应商。

未来可以将相同协议封装为 MCP Server，让模型读取日历、写入提醒或调用更多外部工具，而浏览器端仍保留本地校验和冲突保护。
